# Reproducing the Defense Trilemma empirical results

Every table and figure in Section 10.1 of the paper
(`paper2_neurips.tex`) can be reproduced from commands documented here.
Offline reproduction (using the committed live archives in
`live_runs/`) needs only `numpy` and `matplotlib` and takes **< 15
minutes** of wall clock and **$0**. Full re-collection against the
OpenAI API costs roughly **$45–60** (excluding the canonical saturated
run, which is $~15 by itself).

The source of truth for each number is the `result.json` that the
validator writes alongside every command's output directory. The
per-cell records (each cell's `f(x)`, `D(x)`, `f(D(x))`, distance to
`z*`, ε-robust LHS/RHS, steep/persistent membership) are in the same
JSON.

---

## Prerequisites

- Python 3.10+
- `pip install -e ./trilemma_validator` (or `uv pip install -e .`
  inside the package dir)
- `OPENAI_API_KEY` — **only** needed for live runs (`trilemma
  experiment`, `trilemma pipeline`). All offline analysis of the
  committed archives under `live_runs/` runs without a key.
- Optional, for the live re-collection only:
  [`rethinking-evals`](https://github.com/kingroryg/rethinking-evals)
  checked out somewhere on disk; its path is passed as
  `--rethinking-path`.

Every command below assumes your working directory is the
`trilemma_validator/` package root.

## Canonical saturated run (reproduces Table `tab:smoketests` saturated row, Figure `fig:theory-vs-reality-saturated`)

This is the **only** live run in which Theorem 6.2 is non-vacuously
exercised: 82 filled cells out of 625, 66 strictly unsafe, 10 boundary,
under the canonical identity defense.

Offline (uses the committed archive):

```bash
trilemma validate \
    --archive live_runs/gpt35_turbo_t05_saturated/source_archive.json \
    --tau 0.5 --defense identity \
    --output live_runs/gpt35_turbo_t05_saturated
```

Cost: $0. Wall clock: < 1 min. Output: `result.json`, `report.md`,
`validation.png`, `heatmap.npy`.

Live re-collection:

```bash
trilemma pipeline \
    --rethinking-path /path/to/rethinking-evals \
    --model gpt35_turbo --judge gpt-4.1-2025-04-14 \
    --iterations 300 --seed-prompts 100 \
    --tau 0.5 --defense identity \
    --output live_runs/gpt35_turbo_t05_saturated_repro
```

Cost: ~$15 (5,200 API calls at gpt-3.5-turbo-0125 pricing, plus
gpt-4.1 judge calls). Wall clock: ~130 min. The archive will differ
byte-for-byte from the committed one (MAP-Elites is stochastic; OpenAI
APIs are non-deterministic at temperature > 0) but the headline counts
(`|S_pred| = |S_act| = 66`, `FP_int = 0`) should reproduce within
bootstrap CI.

Figure 4 is regenerated by:

```bash
python scripts/make_paper_figure.py
```

which also regenerates `figures/oblique_theory_vs_reality.pdf`.

The RBF × Matérn-3/2 × Matérn-5/2 crossed with σ ∈ {0.1, 0.2, 0.4}

```python
    tau=0.5,

"
```

Cost: $0 (no API calls; pure numpy GP fits). Wall clock: ~2 min for 9
cells. Produces `sensitivity.json` + per-cell `result.json` files +
the LaTeX table the paper consumes.

## Bootstrap CIs (reproduces Table `tab:ci`)

```bash
trilemma validate \
    --archive live_runs/gpt35_turbo_t05_saturated/source_archive.json \
    --tau 0.5 --defense identity \
    --bootstrap 1000 \
    --output live_runs/gpt35_turbo_t05_saturated_ci
```

Cost: $0. Wall clock: ~1 min. The `result.json` at
`live_runs/gpt35_turbo_t05_saturated_ci/result.json` grows a
`bootstrap` key with `ci_L`, `ci_K`, `ci_ell`, `ci_G` — each as
`{median, lo, hi}` percentile CIs from B=1000 resamples of the
per-pair ratios. Re-render the LaTeX table from these values using
the paper agent's table renderer (the paper consumes
`tables/ci.tex`).

## Seed replication (reproduces Table `tab:seed-replication`)

**Status:** the table label `tab:seed-replication` is listed in the
reproducibility inventory but is **not referenced in
`paper2_neurips.tex`** at this writing. The underlying data is the
output of running `trilemma pipeline` with multiple `--seed-prompts`
values. Example sweep:

```bash
for seed in 7 23 41 97; do
    PYTHONHASHSEED=$seed trilemma pipeline \
        --rethinking-path /path/to/rethinking-evals \
        --model gpt35_turbo --judge gpt-4.1-2025-04-14 \
        --iterations 300 --seed-prompts 100 \
        --tau 0.5 --defense identity \
        --output live_runs/seed_replication/seed_$seed
done
```

Cost: 4 × $15 = **$60**. Wall clock: ~8 h. Each run produces its own
`result.json`; the seed-replication table is the headline numbers
(`|S_act|`, `|S_pred|`, `TP`, `FP_int`, `FN`) aggregated across seeds.
Flag: if Agent 1 lands this table, it should be wired to
`\input{tables/seed_replication.tex}` from `paper2_neurips.tex`.

## Per-table expected outputs and tolerances

All tolerances assume the canonical judge `gpt-4.1-2025-04-14`. With a
different judge, the alignment-deviation surface changes and all
estimates shift — we therefore only promise exact reproduction when
validating the committed archive under `identity`, and
within-bootstrap-CI reproduction under any other configuration.

| Table | Defense | Expected headline | Tolerance |
|---|---|---|---|
| `tab:smoketests` (gpt-5-mini row) | identity | `|S_pred| = |S_act| = 7`, `TP=7`, `FP_int=0`, `FN=0`, `L=G=6.13` | Exact on committed archive. |
| `tab:smoketests` (gpt-3.5-turbo smoke row) | identity | `|S_pred| = |S_act| = 7`, `TP=7`, `FP_int=0`, `FN=0`, `L=G=11.44` | Exact on committed archive. |
| `tab:smoketests` saturated row (in Fig. 4) | identity | `|S_pred|=|S_act|=66`, `TP=66`, `FP_int=0`, `FN=0`, `L=G=23.63` | Exact on committed archive. On a live re-collection, `|S_act|` fluctuates ±5 due to MAP-Elites stochasticity; `FP_int = 0` is theorem-required and **must** reproduce exactly. |
| `tab:live-sweep` (discrete stress) | bounded_step(1..12) | `FP_int = 0` for every row | Theorem-required; **exact reproduction**. `|S_act|` counts fluctuate with the heatmap. |
| `tab:ci` (bootstrap CI) | identity | CI intervals depending on B=1000, seed=0 | Exact on committed archive. Under a different seed, CI endpoints move < 5% for a 1000-resample percentile. |
| `tab:gp-sensitivity` | oblique_gp_smooth over 9 (kernel, σ) cells | `FP_int = 0` for all 9 cells; transversality holds everywhere | Theorem-required (`FP_int = 0`); exact on committed archive. Numerical `L_gp`, `K_emp`, `ell_emp` reproduce to 1 e-9 given fixed seed. |
| `tab:continuous-sweep` | 4 continuous defenses | `FP_int = 0` for all rows; `transv` ✓ for oblique_gp_smooth | Theorem-required; exact on committed archive. |
| `tab:resolution` | oblique_gp_smooth on 13/17/21/25 | `FP_int = 0` at every resolution; transversality holds at 25, may fail at 13 | Theorem-required (when hypotheses hold). Exact on committed archive. |

## Verifying a replication

A replication "matches the paper's numbers within bootstrap CI" iff:

1. **`FP_int = 0` exactly** for every row in every theorem-testing
   table (this is theorem-required, not statistical).
2. **`L`, `K`, `ell`, `G`** each fall inside the reported bootstrap
   CI from `tab:ci` (the CI widths are honest measures of per-pair
   resampling noise).
3. **`|S_act|` fluctuation** up to ±5 cells on a 25×25 saturated run
   (MAP-Elites stochasticity); a larger deviation indicates the
   underlying archive is from a different target model, a different
   judge, or a different iteration budget.

## Disclosure

All live runs in the paper used `gpt-4.1-2025-04-14` as the judge,
`gpt-3.5-turbo-0125` and `gpt-5-mini-2025-08-07` as targets, and the
rethinking-evals MAP-Elites driver with `grid_size=25`. Exact seed,
temperature, and prompt-budget values are recorded in each
`source_archive.json` under `live_runs/`. The oblique GP-smooth
defense uses `alpha_step=0.003`, `sigmoid_steepness=2.0`,
`oblique_angle_deg=89.5°`, `sigma=0.2`, `noise=0.02`, as documented in
`sensitivity.py::run_sensitivity_cell`.
