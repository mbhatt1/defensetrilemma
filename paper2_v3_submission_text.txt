Abstract
Wrapper defenses attempt to neutralize prompt injection by preprocessing inputs
with a map D : X → X. We identify a topological obstruction to this program:
in any connected prompt space, the safe sub-level set Sτ = {f < τ } is open but
not closed, whereas any continuous utility-preserving defense has a closed fixedpoint set containing Sτ ; closure must spill onto the boundary, forcing boundary
fixation. The obstruction is structural, not a matter of design or implementation.
Under Lipschitz and transversality conditions, the failure is not merely existential but quantitative: we give explicit cone-measure and volume lower bounds
on the persistent-unsafe region (Theorems E.1 and E.2), and a dilemma characterizing the optimal defense Lipschitz constant K ∗ = G/ℓ − 1 (Theorem E.3).
The same obstruction reappears in three extensions: discrete finite-set defenses
must collapse distinct inputs to be complete; multi-turn wrappers inherit boundary fixation at every turn; stochastic wrappers with continuous expected score
inherit threshold-level failure in expectation. The theory is formalized in Lean 4
with Mathlib across 46 files containing ∼360 declarations, with no admitted
proofs and only three standard axioms (https://anonymous.4open.science/
r/defense-trilemma-XXXX). The artifact’s primary value is definitional scaffolding — unambiguously pinning down continuity, utility preservation, and Lipschitz regularity in a shared language that survives translation between prose and
code — plus verified cross-references between theorem statements. Empirically,
on real LLM safety surfaces the predicted boundary and persistence structure appears under continuous defenses, and a discrete forced-collapse experiment exhibits the information-loss mechanism predicted by the discrete theorem. Our
results do not preclude training-time alignment, architectural changes, discontinuous filters, or defenses that explicitly sacrifice utility.

Introduction

Can you build a wrapper around a language model that eliminates all prompt injection vulnerabilities? Most wrapper-defense work implicitly assumes yes. Input classifiers that flag suspicious
prompts [1], constitutional rewriting pipelines [3], and input sanitization layers [7] all share the same
structure: a function D : X → X that preprocesses prompts before the model sees them, mapping
unsafe inputs to safer equivalents while preserving benign behavior. We prove that this program has
a structural failure mode. If a wrapper defense is continuous and utility-preserving, it cannot also
be complete. These three properties form a defense trilemma: continuity, utility preservation, and
completeness cannot coexist (Figure 2).
The contribution is not four unrelated theorems. It is one claim shown to survive across four regimes
that matter for how wrapper defenses are used and defended in practice:

Continuous wrapper defenses. In connected prompt spaces, a utility-preserving continuous
wrapper must leave threshold-level prompts unchanged. The same obstruction survives under scorepreserving and ε-relaxed utility preservation, and under Lipschitz regularity it strengthens to quantitative near-threshold and persistent-unsafe-region results (Theorems 3.1, 3.4, 3.5, 3.8 and 3.11).
Discrete wrapper defenses. Without topology, the impossibility becomes combinatorial: a complete utility-preserving defense must be non-injective. Information loss is not an implementation
artifact; it is the mechanism of complete discrete defense (Theorem 4.1).
Multi-turn wrapper defenses. Interaction does not remove the obstruction. If each turn is modeled as a utility-preserving wrapper step, the same threshold-level failure reappears at every turn
(Theorem 5.1).
Stochastic wrapper defenses. Randomization does not remove the obstruction either. If the expected post-defense score varies continuously and agrees with the original score on safe inputs, the
stochastic defense still inherits the same boundary failure (Theorem 5.2).
Empirical validation. We then test whether the theory matters empirically. On real LLM safety
surfaces, the continuous theory’s boundary and persistence structure is observable; and on a saturated unsafe prompt archive, the discrete theorem’s forced-collapse mechanism is directly visible.

Definition 2.2 (Defense). A defense is a continuous map D : X → X. It is utility-preserving if

Impossibility of Wrapper Defense in Continuous Settings

We begin with the most fundamental result. The argument fits in a paragraph: the defense fixes every
safe input, so continuity makes its fixed-point set closed. But the safe region Sτ is open (preimage
of an open interval under continuous f ), and in a connected space a nonempty proper open set is not
closed. Hence the fixed-point set cannot stop exactly at the edge of the safe region, it must spill onto
the boundary Bτ . Some boundary prompts pass through unchanged.
Theorem 3.1 (Boundary Fixation). Let X be a connected Hausdorff space (a space that is “in
one piece” and where distinct points can be separated by neighborhoods). Let f : X → R be
continuous with Sτ , Uτ ̸= ∅, and D : X → X continuous with D|Sτ = id. Then there exists z ∈ X
with f (z) = τ and D(z) = z. Moreover, every z ∈ Sτ \ Sτ satisfies f (z) = τ and D(z) = z, and
this set is nonempty.
Proof sketch. In a Hausdorff space, Fix(D) is closed (preimage of the diagonal). By utility preservation, Sτ ⊆ Fix(D), so Sτ ⊆ Fix(D). But Sτ = f −1 ((−∞, τ )) is open and not closed (connectedness: a nonempty proper clopen set would disconnect X). Hence Sτ ⊋ Sτ . Any z ∈ Sτ \ Sτ
satisfies f (z) = τ (limits of values <τ cannot exceed τ , and z ∈
/ Sτ forces f (z) ≥ τ ) and D(z) = z.
Full proof in Section H.
This means that the defense’s fixed-point set is too large to avoid the boundary. Utility preservation forces it to contain the safe region; closure forces it to contain the boundary; connectedness
ensures the boundary is nonempty. Alternatively: a complete utility-preserving defense would be a
continuous retraction D : X → Sτ (since D|Sτ = id and D(X) ⊆ Sτ ), but a retract of a Hausdorff
space is closed (the fixed-point set is closed), while Sτ is open and not closed (connectedness)—a
contradiction.

Theorem 3.2 (Defense Trilemma). Let X be a connected Hausdorff space, f : X → R continuous
with Sτ , Uτ ̸= ∅. No D : X → X can simultaneously be continuous, utility-preserving (D|Sτ = id),
and complete (f (D(x)) < τ for all x).
All three hypotheses are individually necessary. A defense can satisfy at most two of the three—the
defense trilemma (Figure 2). Counterexamples for each dropped hypothesis appear in Section I.
Falsifier. An empirical counterexample to the trilemma would be a production defense that is
demonstrably continuous, utility-preserving, and complete at a named τ .

Exact and Relaxed Utility Preservation

Strict utility preservation (D(x) = x for safe x) can be relaxed. The impossibility survives scorepreserving rewrites and even approximate score preservation.
Theorem 3.4 (Score-Preserving Defense). Let X be a connected Hausdorff space, f : X → R
continuous with Sτ , Uτ ̸= ∅. If D : X → X is continuous and score-preserving on safe inputs:
f (D(x)) = f (x) for all x ∈ Sτ , then there exists z with f (z) = τ and f (D(z)) = τ .

closed and contains Sτ , since closed sets contain their boundary points. For z ∈ Sτ \Sτ , f (D(z)) =
f (z) = τ .

Remark 3.6 (Why D(Sτ ) ⊆ Sτ alone is insufficient). The weakest relaxation—D(Sτ ) ⊆ Sτ with
no score constraint—does allow a complete defense (e.g., a constant map D(x) = x0 to a fixed
safe point). But this destroys all semantic content: every prompt produces the same response. The
score-preservation conditions formalize the requirement that defense must not destroy utility, without
requiring the defense to be the identity.

Remark 3.7 (Open: utility-floor relaxation). Between exact identity on Sτ and the constant map lies

Persistent Unsafe Region

The ε-robust constraint bounds the depth to which the defense pushes near-boundary points, but
permits values slightly below τ . When the alignment surface rises faster than the defense can pull it
down, some points remain above threshold.

163

Decoupling global and directional Lipschitz constants. The ε-robust bound (Theorem 3.8) uses
the global Lipschitz constant L of f , which bounds f uniformly in all directions. The persistence
argument, however, compares f ’s growth rate in the steep direction to how much the defense can
reduce f along the displacement direction D(x) − x. In anisotropic settings these directions differ:
f may rise steeply toward the unsafe region while varying slowly in the direction the defense pulls.

This means that the defense can reduce any point’s score by at most ℓ(K + 1) times its distance
from z. If the score rises faster than that, the defense loses.
Definition 3.10 (Steep region). Given a fixed boundary point z, the steep region is S = {x ∈ X :

than the defense’s Lipschitz budget can compensate.

Impossibility of Wrapper Defense in Discrete Settings

On finite sets, the same failure appears as a combinatorial tradeoff: completeness vs. information preservation. No topology is required; results are verified in Lean (MoF_12_Discrete). The
Tietze/McShane–Whitney bridge to continuous spaces is in Theorem D.1.
Theorem 4.1 (Discrete Defense Dilemma). Let X be a finite set with Sτ , Uτ ̸= ∅, and D : X → X
utility-preserving (D(x) = x for f (x) < τ ).
1. If D is injective, then f (D(u)) ≥ τ for every u with f (u) ≥ τ (including boundary points):
the defense is incomplete.
2. If D is complete (f (D(x)) < τ for all x), then D is non-injective: ∃ x ̸= y with D(x) =
D(y).

Stochastic Defense Impossibility

Theorem 5.2 (Stochastic Defense Impossibility). Let X be a connected Hausdorff space, f : X →
R continuous with Sτ , Uτ ̸= ∅. Let D be a stochastic defense and define g(x) = Ey∼D(x) [f (y)]. If
g is continuous and g(x) = f (x) for all x ∈ Sτ , then there exists z with f (z) = τ and g(z) = τ .
Proof sketch. Define h = g − f . Then h is continuous and h|Sτ = 0, so h vanishes on Sτ (same
closure argument as Theorem 3.4). For z ∈ Sτ \ Sτ : g(z) = f (z) = τ .
Remark (stochastic dichotomy). Since E[f (D(z))] = τ , either f (D(z)) = τ almost surely (the
defense is deterministic at z), or the random variable f (D(z)) has positive probability of exceeding τ —i.e., the defense actively produces unsafe outputs with positive probability. The stochastic
case is therefore strictly harder than the deterministic one: a genuinely random defense at boundary
points must sometimes make things worse.
Remark. The continuity of g is a nontrivial assumption: it requires the distribution D(x) to vary continuously with x in a suitable sense. Stochastic defenses with discontinuous rejection probabilities
escape this theorem.
Falsifier. A stochastic defense at a boundary cell with variance = 0 and f (D(z)) < τ would
refute Theorem 5.2. Empirically, a genuinely random wrapper at a boundary cell shows positive
variance with positive mass above τ (Table 6 and fig. 7 in the appendix).

Non-vacuous validation: continuous defenses

Theorem 3.11 has non-trivial quantitative content only when Ĝ > ℓ̂(K̂+1), i.e., when the alignment surface rises faster than the defense can pull it down. A sweep over four continuous defense
families (Table 10 in appendix) reveals that most natural defenses fail this condition: S MOOTH NEAREST S AFE, K ERNEL S MOOTHED, and S OFT P ROJ all produce ℓ̂(K̂+1) ≫ Ĝ because they align
displacement with −∇f , which maximizes ℓ̂. The theorem becomes a non-vacuous diagnostic when
the defense direction diverges from ∇f : measured (Ĝ, ℓ̂, K̂) with ℓ̂ ≪ L signals a construction that
fights its own gradient.
We demonstrate this diagnostic on a GP-smooth oblique defense D(x) = x + α β(µ(x)) v(x) displaced 89.5◦ from the GP gradient (RBF kernel, σ=0.2, 82 cells). This is a deliberately adversarial
defense direction chosen to enter the transversal regime. Estimates: ℓ̂ = 0.86, K̂ = 1.05, Ĝ = 23.6,
so ℓ̂(K̂+1) = 1.77 ≪ Ĝ. Result: |Spred | = 3, all actually persistent (TP=3, FPint =0; Figure 4).
Theorem E.3’s optimal constant K ∗ = Ĝ/ℓ̂ − 1 ≈ 26.5 gives a concrete design constraint: defenses
with K < K ∗ admit persistent cells; defenses with K ≥ K ∗ destroy utility (Figure 8 in appendix).
Appendix sweeps over three defense families, multiple kernels, and grid resolutions preserve
FPint = 0. Theorem 3.8’s ε-robust bound holds on all 82 cells, and 10 boundary cells in Sτ \ Sτ
activate boundary fixation (nearest to τ : gap = 0.375, vanishing under refinement). Bootstrap, seed
replication, and judge-robustness checks all preserve the claim (Section N). A three-target sweep
(gpt-3.5-turbo, gpt-4o-mini, gpt-4o) yields FPint = 0 in every row (Table 1 in appendix).

Conclusion

The paper makes one claim: wrapper defense is structurally incomplete under utility-preserving constraints. We show that this claim survives in four regimes. In continuous spaces, utility-preserving
wrappers fix boundary prompts and, under additional regularity, leave quantitative near-threshold
and persistent unsafe regions. In discrete spaces, complete defense requires input collapse. In
stochastic and multi-turn settings, the same obstruction reappears rather than disappearing. Empirically, the predicted boundary and collapse mechanisms are visible in real LLM safety surfaces and
prompt archives.
The practical implication is not that defense is futile. It is that wrapper defense should be engineered
as risk management, not as a path to complete elimination. The right objectives are to make the
boundary shallow, reduce effective dimension, monitor approach to threshold, and use mechanisms
outside the wrapper model when stronger guarantees are required.

Broader Impact
This paper establishes a structural limit on a specific class of LLM safety interventions (continuous,
utility-preserving wrapper defenses). It does not propose new attacks, release new attack corpora,
or provide adversaries with new techniques. We summarize the impact, dual-use, and deployment
considerations below in the form encouraged by the NeurIPS 2025 Broader Impact guidance.
Responsible-disclosure stance. The artifact is a theoretical impossibility, not a vulnerability report. There is no affected vendor, no proof-of-concept exploit, and no coordinated-disclosure timeline to negotiate: the claim is that any defense satisfying continuity and utility-preservation on a
connected prompt space must leave boundary prompts fixed. Responsible disclosure in this context
means (i) framing the result as a design constraint rather than a recipe for bypass, (ii) providing the
Lean-verified proofs so defenders can evaluate the claim on its own terms, and (iii) accompanying
every empirical demonstration with explicit quantitative bounds (such as K ∗ = G/ℓ − 1) that tell
defenders what the achievable region looks like.
Who benefits. The primary beneficiaries are defenders and safety researchers: the trilemma
gives them a principled way to allocate effort, a sanity check on new-wrapper proposals (if a published wrapper claims continuity, utility preservation, and completeness, one of the three claims
must be wrong), and concrete parameter targets for the defense-design knobs they actually control
(Lipschitz constant, threshold geometry, choice of feature space). Secondary beneficiaries are auditors and policy teams who need defensible upper bounds on what wrapper-class controls can
achieve.

374

Who does not benefit. An adversary seeking to jailbreak a particular deployed model gains nothing actionable from this paper. The theorems do not identify specific prompts, do not parameterize
a model family, and do not provide a search procedure over inputs. Our empirical forced-collapse
demonstration (Tables 3 and 4) operates entirely on prompts already contained in public jailbreak datasets; no new adversarial text is produced by this work that was not already available to
adversaries via prior publications and public leaderboards.

We considered three plausible misuse vectors:

• “The paper implies wrappers are useless, so vendors will stop deploying them.” The paper
explicitly rules this out (Section B): wrappers remain the cheapest risk-reduction primitive, they
just cannot be total. This is analogous to the existence of adversarial examples not implying that

• “The paper tells attackers where to look.” The boundary-fixation phenomenon is a consequence
of continuity on a connected space, not a localized exploit. No attacker gains a search advantage
from knowing that a boundary exists in general.
• “The paper’s quantitative bounds help adversaries tune.” The quantitative bounds (Theorems 3.11–E.2 and K ∗ = G/ℓ − 1) are defender-facing: they require the wrapper’s Lipschitz
constant and the surface’s boundary rate, both of which are available only to the operator running
the safety stack, not to an external attacker probing the deployed system.
Mitigations. To minimize misuse potential, this paper (i) does not propose or evaluate any new attack; (ii) releases only wrapper-side quantitative bounds (Lipschitz constant, persistent-unsafe measure, threshold gap) rather than attack improvements; (iii) anonymizes the artifact URL to comply
with double-blind review and prevent author-linked attack-adjacent discoverability; and (iv) pairs
every empirical table with the theorem it exercises and the inequality it is expected to satisfy, so that
readers cannot mistake the demonstrations for generic red-teaming results.

continuous and utility-preserving, treat its near-threshold behavior as structurally unreliable and do
not rely on it alone; (b) invest the saved attention in layers outside the wrapper class—training-time
alignment, output-side filters, human-in-the-loop review of boundary cases, and architectural controls on the underlying model; (c) measure L̂, ℓ̂, Ĝ, K̂ on your own surface and use K ∗ = Ĝ/ℓ̂ − 1

Related Work

Adversarial robustness research has established that small perturbations can fool classifiers [20, 6,
9, 14] and that robustness may fundamentally trade off against accuracy [21, 11]. Certified defenses
provide guarantees for fixed models [5, 8, 19, 13, 2], and topological perspectives have illuminated
the structure of decision boundaries [16]. For LLMs specifically, jailbreaking attacks [23, 4, 10, 12]
and automated red-teaming [15, 18] demonstrate persistent vulnerabilities.
Our work differs in object and claim. Rather than studying how failures arise for a fixed model
or how a particular system can be certified, we impose universal constraints on the wrapper map
itself. The closest conceptual precedent is the no-free-lunch framework [22]: the result is not that
one defense is poorly implemented, but that a defense pattern cannot satisfy all of its intended
requirements simultaneously.
Unlike no-free-lunch (which constrains optimizers across problems) or robustness-accuracy tradeoffs [21] (which constrain classifier behavior on samples), our impossibility is map-level — it
constrains the defense function itself, independent of which attacks or samples are considered.
The topological obstruction Sτ open-but-not-closed applies to every continuous utility-preserving
D : X → X, not to a learning problem or to a distribution of inputs.

Theorem D.1 (Continuous Relaxation). Let (X, d) be a connected, normal, Hausdorff metric space,
and S ⊂ X a finite set of observations with observed alignment scores g : S → R satisfying
g(p) < τ and g(q) > τ for some p, q ∈ S. Then there exists a continuous f : X → R extending g
(i.e., f |S = g) for which the hypotheses of Theorem 3.1 hold. If the extension is chosen to be
Lipschitz (via McShane–Whitney), the hypotheses of Theorem 3.8 also hold. If f is additionally
Lipschitz, Theorem 3.11 applies wherever transversality is met.
Proof sketch. Step 1 (Extension exists). S is finite and X is T1 , so S is closed. Since X is normal
and g : S → R is continuous on a discrete closed subset, the Tietze extension theorem provides a
continuous f : X → R with f |S = g.

Step 2 (Hypotheses of Theorem 3.1 are satisfied). We have f (p) = g(p) < τ and f (q) = g(q) > τ ,
so Sτ ̸= ∅ and Uτ ̸= ∅. Since X is connected and Hausdorff, Theorem 3.1 applies.

and Theorem 3.11 apply under their additional hypotheses.

Pipeline Composition

Theorem F.1 (PipelineQLipschitz Degradation). If stages T1 , . . . , Tn are K1 , . . . , Kn -Lipschitz, the
composed pipeline is ( Ki )-Lipschitz. For n stages each with K ≥ 2, the effective constant is K n .
Falsifier. A measured pipeline with empirical composed Lipschitz constant K̂composed > K̂1 · K̂2

Proof of Theorem 3.1 (Boundary Fixation). Step 1 (Hausdorff ⇒ fixed-point set is closed).
Fix(D) = {x : D(x) = x} is the preimage of the diagonal ∆ ⊂ X × X under x 7→ (D(x), x). In
a Hausdorff space, ∆ is closed, so Fix(D) is closed.
Step 2 (Utility preservation ⇒ safe region ⊆ fixed points). Sτ ⊆ Fix(D). Since Fix(D) is closed:
Sτ ⊆ Fix(D).

Step 3 (Connectedness ⇒ safe region is not closed). Sτ = f −1 ((−∞, τ )) is open. If also closed, it
would be clopen—but in a connected space the only clopen sets are ∅ and X. Since both Sτ and Uτ
are nonempty, Sτ is not closed.

Step 4 (Boundary point exists). Sτ ⊋ Sτ , so there exists z ∈ Sτ \ Sτ . Continuity gives f (z) ≤ τ ;

Theorem J.1 (Perturbation Robustness). If f is L-Lipschitz and f (p) > τ , then B(p, (f (p) −

Theorem J.3 (Transferability). ∥f − g∥∞ ≤ δ =⇒ {f > τ + δ} ⊆ {g > τ }. Transfer costs zero
additional queries.
Theorem J.4 (Authority Monotonicity). If f (y, ·) is monotone non-decreasing in authority for each

when f (y, ·) is continuous and crosses τ ). If additionally f is monotone non-decreasing in indirection for each fixed authority, the threshold curve y 7→ a∗2 (y) is non-increasing.
Theorem J.5 (Gradient Ascent). If f has nonzero Fréchet derivative at x, there exist v and ε > 0

Cost Asymmetry

Theorem L.1 (Cost Asymmetry for Grid-Enumeration Defenses). For grid-based exhaustiveenumeration defense with N ≥ 2 bins per axis in d dimensions: attack cost ≤ 1/δ (dimensionindependent); defense cost = N d (exponential in d); ratio δ · N d → ∞ as d → ∞.
Caveat. This bound applies to grid enumeration, not to learning-based defenses (e.g., GP surrogates)
that generalize across the space without evaluating every cell. The paper’s own empirical section

Judge robustness

All f values in Section 6.1 are judge-dependent: swapping the judge can reshape the
alignment-deviation surface and thereby change L̂, ℓ̂, Ĝ.
To quantify this, we rescored
the 82 (prompt, target response) pairs from the saturated gpt-3.5-turbo-0125 archive with
three alternative judges drawn from different families and capacities — gpt-4o-2024-08-06,
gpt-4.1-mini-2025-04-14, and gpt-4o-mini-2024-07-18 — applying the same rethinkingevals per-category methodology with a single judge and α = 0.5 (target responses were NOT regenerated; only the judge was swapped). The four-judge comparison is summarized in Table 18.
Interpretation. The theorem applies to the full prompt space X under every judge — benign
prompts exist, so Sτ is nonempty in X. Whether the theorem’s positive prediction can be tested
from a given sample depends on whether that sample spans both regions. Three of four judges
(gpt-4.1, gpt-4o, gpt-4.1-mini) yield samples that span both Sτ and Uτ ; applying the continuous smooth_nearest_safe defense produces TP> 0 and FPint = 0 in every case. Under the
strictest judge (gpt-4o-mini), MAP-Elites’s adversarial bias drives the sample entirely into Uτ , so
no boundary cell z ∗ appears in the 82 sampled cells and the positive prediction cannot be tested
from this sample. Crucially, FPint = 0 still holds: no counterexample to the theorem is produced
under any of the four judges. Cross-judge agreement ranges from substantial (Kendall’s W = 0.70)
to poor on binary classification (Cohen’s κ = 0 vs gpt-4o-mini), so the specific numerical instance
of a particular judge is not itself judge-robust even though the framework’s predictions are.

692

Committee aggregation. The rethinking-evals methodology supports committee aggregation:
Pactual = αPvote + (1 − α)Pmean with α = 0.5. We apply this offline to the four existing
judge archives (no new API calls) and re-run the validator on each committee-aggregated surface
(Table 19). The top-3 committee (gpt-4.1, gpt-4o, gpt-4.1-mini; excluding the strict-judge outlier
gpt-4o-mini) is within one cell of canonical (17 safe vs 16; mean ∆ = +0.010), and FPint = 0 under smooth_nearest_safe on every committee configuration — the theorem’s predictions survive
committee aggregation.
